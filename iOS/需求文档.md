# 需求文档

## 1. 应用概述

### 1.1 应用名称
iOS风格图片浏览器

### 1.2 应用描述
一个采用iOS系统视觉风格的Web图片浏览器应用，支持从指定接口获取图片列表、本地缓存、单张/批量操作、个性化主题设置、幻灯片播放、URL参数定位预览等功能。界面采用圆角、阴影、玻璃拟态、动效等iOS视觉元素，提供流畅的交互体验。

## 2. 用户与使用场景

### 2.1 目标用户
需要浏览和管理在线图片资源的用户

### 2.2 核心使用场景
- 用户启动应用时查看服务连接状态
- 用户浏览从服务端获取的图片列表
- 用户通过URL参数直接定位并预览指定ID图片
- 用户对单张或多张图片进行操作（旋转、下载、分享等）
- 用户使用幻灯片模式自动播放图片
- 用户根据个人喜好调整应用主题和界面样式

## 3. 页面结构与功能说明

### 3.1 页面结构

```
应用根目录
├── 服务状态检查页
├── 主页面
│   ├── 图片展示区
│   ├── 图片预览模式
│   ├── 幻灯片播放模式
│   ├── 单张图片操作工具栏
│   ├── 批量操作工具栏
│   └── 设置面板
└── 官网导航链接
```

### 3.2 服务状态检查页

#### 3.2.1 功能说明
- 应用启动时自动进入该页面
- 显示iOS风格加载动画（圆形进度指示器、渐变色彩、流畅动效）
- 检测并显示网络延迟（毫秒单位，实时更新）
- 检测并显示连接网速（Mbps单位，实时更新）
- 获取并显示当前IP地址（卡片式呈现）
- 调用健康检查接口 GET https://memories-api.mrcwoods.com/health，返回 { \"ok\": true } 表示连接成功
- 连接成功后显示成功动效，自动跳转至主页面
- 页面采用iOS风格视觉设计：毛玻璃背景、圆角卡片、柔和阴影、渐变色彩、平滑过渡动效
- 状态检查卡片采用细腻的玻璃拟态与高光效果
- 暗色模式下卡片使用深色渐变背景，加载条使用柔和渐变

### 3.3 主页面

#### 3.3.1 图片展示区
- 从接口 https://memories-api.mrcwoods.com/images?after_id=0 获取图片URL列表
- 接口返回格式：{ \"data\": [{\"id\", \"url\", \"uploaded_at\"}], \"next_after_id\", \"limit\": 20 }
- 每页最多显示20条图片
- 逐个URL加载图片并缓存至本地
- 重复图片不显示
- 图片卡片不显示ID标签
- 图片卡片采用iOS风格设计：圆角、优化后的阴影效果、悬停动效、加载骨架屏
- 图片卡片选中态具有明显视觉反馈（边框高亮、阴影增强）
- 图片加载占位状态显示骨架屏动画
- 图片加载错误状态显示占位图与错误提示
- 提供「加载更多」按钮，点击后使用 next_after_id 参数获取下一页数据
- 默认点击图片卡片进入图片预览模式，增加防误触机制
- 长按图片卡片进入选择模式，长按时提供触摸反馈（震动或视觉反馈），可多选图片进行批量操作
- 图片卡片点击热区覆盖整个卡片区域

#### 3.3.2 图片预览模式
- 点击图片卡片后进入全屏预览模式
- 支持双指捏合放大缩小图片
- 提供「上一张」「下一张」按钮切换图片（iOS风格圆形按钮，半透明背景，点击热区扩大）
- 支持左右滑动切换图片（平滑过渡动效）
- 支持键盘方向键（左右）切换图片
- 显示当前图片序号与总数（顶部居中显示）
- 提供唯一关闭按钮（右上角，iOS风格圆形按钮，半透明背景，点击热区扩大），点击退出预览模式
- 预览界面采用iOS风格：毛玻璃背景、平滑缩放动效、渐入渐出过渡
- 减少预览弹窗的触发误操作

#### 3.3.3 幻灯片播放模式
- 提供「幻灯片播放」按钮启动幻灯片模式（iOS风格按钮设计）
- 进入全屏播放状态，自动轮播当前页面所有图片
- 提供暂停/继续播放按钮（iOS风格圆形按钮，半透明背景，点击热区扩大）
- 提供退出按钮返回主页面（右上角，iOS风格圆形按钮，点击热区扩大）
- 播放过程中支持手动切换上一张/下一张
- 播放界面采用iOS风格：平滑切换动效、渐入渐出过渡

#### 3.3.4 URL参数定位预览功能
- 支持通过URL参数 /gallery?id=123 直接定位并打开指定ID图片预览
- 应用启动时检测URL参数，若存在id参数则自动加载对应图片并进入预览模式
- 若指定ID图片不存在或加载失败，显示提示信息

#### 3.3.5 单张图片操作工具栏
- 旋转：旋转当前选中图片
- 上下翻转：垂直翻转当前选中图片
- 右翻转：水平翻转当前选中图片
- 分享：分享当前选中图片
- 下载：直接触发浏览器下载当前选中图片至本地，兼容跨域图床场景（通过fetch获取图片Blob数据后创建下载链接）
- 复制图片URL：复制当前选中图片的URL至剪贴板
- 放大：放大当前选中图片
- 缩小：缩小当前选中图片
- 还原：将当前选中图片恢复至原始状态
- 拖拽：支持拖拽移动图片位置
- 工具栏采用iOS风格：毛玻璃背景、圆角按钮、图标设计、悬停动效、细腻的玻璃拟态与高光效果
- 按钮点击热区扩大，统一按钮间距与字号层级
- 暗色模式下主按钮与强调色使用柔和渐变

#### 3.3.6 批量操作工具栏
- 全选：选中当前页面所有图片
- 批量分享：分享所有选中的图片
- 批量下载：直接触发浏览器下载所有选中的图片至本地，兼容跨域图床场景（通过fetch获取图片Blob数据后创建下载链接）
- 批量复制URL：复制所有选中图片的URL至剪贴板
- 工具栏采用iOS风格：毛玻璃背景、圆角按钮、图标设计、悬停动效、细腻的玻璃拟态与高光效果
- 按钮点击热区扩大，统一按钮间距与字号层级
- 移动端增大图标间距，便于触摸操作
- 暗色模式下主按钮与强调色使用柔和渐变

## 4. 业务规则与逻辑

### 4.1 图片去重规则
- 根据图片URL判断是否重复
- 重复图片不在展示区显示

### 4.2 图片缓存规则
- 图片加载成功后缓存至本地
- 再次访问相同URL时优先使用本地缓存

### 4.3 翻页逻辑
- 首次加载使用 after_id=0
- 后续翻页使用接口返回的 next_after_id 参数
- 每页最多返回20条数据

### 4.4 操作状态保持
- 单张图片的旋转、翻转、缩放状态仅在当前会话有效
- 点击「还原」按钮恢复至原始状态

### 4.5 主题与设置持久化
- 用户选择的主题、界面模式、字体、字号等设置需持久化保存
- 下次启动应用时自动应用上次设置

### 4.6 选择模式触发规则
- 默认点击图片卡片进入预览模式
- 长按图片卡片进入选择模式，可多选图片
- 选择模式下点击图片卡片切换选中状态
- 退出选择模式后恢复默认点击行为

### 4.7 图片预览切换规则
- 支持按钮点击、左右滑动、键盘方向键三种方式切换图片
- 切换至第一张时「上一张」按钮禁用
- 切换至最后一张时「下一张」按钮禁用

### 4.8 幻灯片播放规则
- 自动轮播间隔时间固定
- 播放至最后一张后循环至第一张继续播放
- 暂停后可手动切换图片，继续播放后从当前图片开始

### 4.9 URL参数定位规则
- 检测URL参数id，若存在则优先加载对应图片
- 加载成功后自动进入预览模式
- 加载失败则显示提示信息并返回主页面

### 4.10 iOS视觉风格规则
- 界面元素采用圆角设计
- 卡片、按钮、面板使用优化后的阴影效果
- 部分区域使用玻璃拟态效果（毛玻璃背景）
- 交互操作添加平滑动效（渐入渐出、缩放、滑动过渡）
- 按钮采用半透明背景、悬停动效
- 加载状态使用骨架屏或iOS风格进度指示器
- 色彩采用多色浅色渐变风格：主色、强调色、背景色、卡片色等通过柔和、明快的浅色渐变过渡呈现
- 状态检查页、图片卡片、按钮、设置面板等位置体现渐变背景与高光细节
- 浅色模式下的渐变效果优先且明显，暗色模式保持可用
- 暗色模式下为每套主题定义暗色版渐变与色板，降低饱和度并提升对比度，避免背景与文字对比不足
- 暗色模式下卡片使用深色渐变背景而非纯黑
- 暗色模式下主按钮、强调色、加载条使用柔和渐变
- 顶栏、底栏、工具栏、状态检查卡片采用细腻的玻璃拟态与高光效果
- 统一按钮、输入框、标签的间距与字号层级

### 4.11 移动端适配规则
- 小屏设备下调整布局与元素尺寸
- 增加触摸区域大小，便于操作
- 底部工具栏不遮挡内容区域
- 移动端批量操作工具栏增大图标间距

### 4.12 图片下载规则
- 点击下载按钮时，通过fetch获取图片Blob数据
- 创建临时下载链接并触发浏览器下载行为
- 兼容跨域图床场景，避免直接打开图片链接
- 下载完成后释放临时链接资源

### 4.13 图片预览界面关闭按钮规则
- 图片预览界面仅显示唯一关闭按钮（右上角）
- 禁止出现重复关闭按钮

### 4.14 交互优化规则
- 减少预览弹窗的触发误操作
- 优化长按时触摸反馈（震动或视觉反馈）
- 提高按钮与卡片的点击热区
- 移动端增大批量操作工具栏图标间距
- 主题切换、模式切换增加平滑过渡动效
- 优化无数据、加载失败等空状态提示

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 健康检查接口返回非 { \"ok\": true } | 显示连接失败提示，提供重试按钮 |
| 图片列表接口请求失败 | 显示加载失败提示，提供重试按钮 |
| 图片URL加载失败 | 显示占位图，标记该图片加载失败 |
| next_after_id 为空 | 表示已加载全部图片，隐藏「加载更多」按钮 |
| 批量操作时未选中任何图片 | 提示用户至少选择一张图片 |
| 网络断开 | 显示网络异常提示 |
| 复制URL失败 | 提示复制失败 |
| 下载图片失败 | 提示下载失败 |
| 图片元数据接口返回404或无响应体 | 捕获异常，避免报错 Failed to execute 'json' on 'Response': Unexpected end of JSON input，显示「元数据获取失败」提示 |
| URL参数id对应图片不存在 | 显示「图片不存在」提示，返回主页面 |
| URL参数id对应图片加载失败 | 显示「图片加载失败」提示，返回主页面 |
| 幻灯片播放时网络异常 | 暂停播放，显示网络异常提示 |
| 预览模式下切换至不存在的图片 | 显示占位图，提示图片加载失败 |
| 跨域图床下载失败 | 提示下载失败，建议用户手动保存图片 |
| 无数据状态 | 显示优化后的空状态提示 |
| 加载失败状态 | 显示优化后的加载失败提示 |

## 6. 验收标准

1. 启动应用，服务状态检查页显示iOS风格加载动画、网络延迟、连接网速、当前IP，健康检查接口返回成功后显示成功动效并自动跳转至主页面
2. 主页面自动从接口 https://memories-api.mrcwoods.com/images?after_id=0 获取图片列表，逐个加载并显示图片，图片卡片采用iOS风格设计且不显示ID标签
3. 点击图片卡片进入预览模式，界面仅显示唯一关闭按钮（右上角），支持双指捏合放大缩小，支持按钮、左右滑动、键盘方向键切换上一张/下一张
4. 长按图片卡片进入选择模式，可多选图片，点击批量下载按钮，所有选中图片直接触发浏览器下载至本地（兼容跨域图床）
5. 点击「幻灯片播放」按钮，进入全屏播放模式，图片自动轮播，点击暂停按钮暂停播放，点击退出按钮返回主页面
6. 访问 /gallery?id=123，应用自动加载ID为123的图片并进入预览模式
7. 调用图片元数据接口时，若返回404或无响应体，不报错 Failed to execute 'json' on 'Response': Unexpected end of JSON input，显示「元数据获取失败」提示
8. 在移动端设备上访问应用，界面布局适配小屏，触摸区域足够大，底部工具栏不遮挡内容
